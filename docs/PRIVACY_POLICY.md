# Privacy Policy for Header Override

Last updated: July 20, 2026

Header Override lets users create local rules that override HTTP headers and cookies for URLs they choose.

## Data the extension handles

Header Override stores the rules you create in your browser's local extension storage. These rules may include:

- Header names.
- Header values.
- Cookie names.
- Cookie values.
- Cookie attributes.
- Cookie domains and paths.
- URL filter patterns.
- Whether each rule is enabled.

The extension uses these rules only to configure the browser's request rules API, write request cookies you configured, and apply the header or cookie changes you requested.

## Data collection and sharing

Header Override does not send your rules, browsing activity, website content, or any other personal data to the developer or to any third party. The extension does not use analytics, advertising, tracking, remote code, or external servers.

## Data storage and deletion

Rules are stored locally in your browser using extension storage. You can edit or delete rules from the extension popup. Uninstalling the extension removes the locally stored extension data according to the browser's normal extension storage behavior.

## Permissions

Header Override requests access to all URLs so that user-created rules can apply to the websites and requests selected by the user's URL filters. It requests storage permission so rules can be saved locally.

## Contact

For privacy questions, use the support contact listed on your browser's extension store listing.
